OVERVIEW
========

Welcome! In this assignment, you'll learn one of the most fundamental concepts 
in programming: functions that receive information and return it back. Think of 
these functions as simple "echo chambers" or "mirrors" that take what you give 
them and send it right back to you.

You'll be creating two simple functions:
  1. id() - A function that returns exactly what you give it
  2. getLength() - A function that tells you how long a string or array is

These might seem simple, but they teach you essential programming patterns that 
you'll use in every project you build!


================================================================================
KEY CONCEPT #1: WHAT ARE FUNCTIONS?
================================================================================

A function is like a recipe in a cookbook:
  - You write down the steps once
  - You can use those steps many times
  - Each time you use it, you might use different ingredients (arguments)
  - The function does its job and gives you back a result (return value)

Example analogy: A vending machine
  - You insert money (input/argument)
  - The machine processes your request
  - You get your snack (output/return value)


================================================================================
KEY CONCEPT #2: ARGUMENTS (INPUTS)
================================================================================

Arguments are the information you give to a function. They're like parameters 
in a recipe.

When you create a function, you specify what arguments it accepts by naming 
them inside the parentheses:

    function myFunction(argument1, argument2) {
      // function body
    }

Then when you CALL (use) the function, you provide actual values:

    myFunction(42, "hello")

In this call:
  - argument1 receives the value 42
  - argument2 receives the value "hello"


================================================================================
KEY CONCEPT #3: RETURN VALUES (OUTPUTS)
================================================================================

The return statement is how a function sends information back to you.

Example:
    function add(a, b) {
      return a + b;
    }
    
    let result = add(3, 4);  // result now holds 7

Important: When a function uses 'return', it STOPS executing right there. 
Nothing after the return statement runs.

    function example() {
      return 5;
      console.log("This never runs!");  // This line is unreachable
    }


================================================================================
THE ASSIGNMENT: FUNCTION #1 - id()
================================================================================

Purpose: 
  The id() function is like an identity function - it takes one argument and 
  returns that exact same argument unchanged.

Why learn this?
  This teaches you the basics:
    - How to define a function
    - How to accept an argument
    - How to return that argument
  
  It's the simplest building block. Many advanced concepts build on this!

How it should work:
    id(5)           returns 5
    id("hello")     returns "hello"
    id([1, 2, 3])   returns [1, 2, 3]
    id(true)        returns true

The key insight: The function doesn't change anything - it just passes the 
argument straight through. This is actually useful! (We'll see why in more 
advanced lessons.)

Implementation:
    function id(argument) {
      return argument;
    }

THE ASSIGNMENT: FUNCTION #2 - getLength()
================================================================================

Purpose:
  The getLength() function takes either a string or an array and returns how 
  many characters (for strings) or how many items (for arrays) it contains.

Why learn this?
  This teaches you:
    - How to work with the .length property
    - How to create a function that works with multiple data types
    - How to measure and get information about data

How it should work:
    getLength("hello")      returns 5 (h-e-l-l-o)
    getLength("xyz")        returns 3
    getLength([1, 2, 3])    returns 3 (3 items in array)
    getLength([])           returns 0 (empty array)
    getLength("")           returns 0 (empty string)

Understanding .length:
  Both strings and arrays have a built-in property called .length that tells 
  you their size. You access it with a dot:
  
    "hello".length    → 5
    [1, 2, 3].length  → 3

The function just needs to:
  1. Accept an argument (either string or array)
  2. Access its .length property
  3. Return that length

Implementation:
    function getLength(stringOrArray) {
      return stringOrArray.length;
    }

================================================================================
IMPORTANT CONCEPTS TO REMEMBER
================================================================================

1. The .length Property
   - Both strings and arrays have this built-in property
   - Access it with a dot: stringName.length or arrayName.length
   - It returns a number showing the size

2. The difference between arrays and strings
   - Strings: sequences of characters, use quotes: "hello"
   - Arrays: sequences of items, use brackets: [1, 2, 3]
   - Both can be measured with .length

3. Function naming conventions
   - Use camelCase: myFunction, getLength, getElementById
   - Not snake_case: my_function (that's Python style)
   - The name should describe what the function does

4. Return statement
   - It sends a value back to wherever the function was called
   - Execution stops at return - nothing after it runs
   - Every function returns something (even if you don't specify, it returns undefined)

COMMON BEGINNER MISTAKES (AND HOW TO AVOID THEM)
================================================================================

Mistake #1: Forgetting the 'return' keyword
  ✗ WRONG:
    function id(x) {
      x;  // This just sits there, doesn't return anything
    }
  
  ✓ RIGHT:
    function id(x) {
      return x;
    }

Mistake #2: Confusing console.log() with return
  ✗ WRONG:
    function id(x) {
      console.log(x);  // This prints to screen, but doesn't return
    }
  
  ✓ RIGHT:
    function id(x) {
      return x;  // This actually returns the value
    }
  
  Note: console.log() is for debugging (seeing output). 
        return is for sending data to other parts of your code.

Mistake #3: Forgetting parentheses when calling a function
  ✗ WRONG:
    let result = id;  // This just gets the function itself, doesn't call it
  
  ✓ RIGHT:
    let result = id(5);  // The () actually calls the function

Mistake #4: Writing return as "returns" or "return;"
  ✗ WRONG:
    function id(x) {
      returns x;  // Wrong keyword
    }
  
  ✓ RIGHT:
    function id(x) {
      return x;  // Correct
    }

// FUNCTION #1: id()
// Purpose: Takes one argument and returns it unchanged
// This is the simplest function - an identity function!
// 
// How it works:
//   - Accepts any value as an argument
//   - Returns that exact same value
//   - Doesn't change or modify anything
//
// Examples:
//   id(42)                  → 42
//   id("hello")             → "hello"
//   id([1, 2, 3])           → [1, 2, 3]
//   id(true)                → true

function id(argument) {
  return argument;
}


// FUNCTION #2: getLength()
// Purpose: Takes a string or array and returns its length
// This function accesses the built-in .length property
//
// How it works:
//   - Accepts a string or array as an argument
//   - Accesses the .length property of that argument
//   - Returns the length as a number
//
// Examples:
//   getLength("hello")      → 5
//   getLength("xyz")        → 3
//   getLength([1, 2, 3])    → 3
//   getLength([])           → 0
//   getLength("")           → 0

function getLength(stringOrArray) {
  return stringOrArray.length;
}
