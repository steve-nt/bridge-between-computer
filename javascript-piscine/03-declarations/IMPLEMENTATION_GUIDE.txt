2. KEY CONCEPTS EXPLAINED
================================================================================

A. WHAT ARE VARIABLES?
Think of variables like labeled containers that hold information. Just like you 
might have a box labeled "Favorite Books" that contains your books, a variable 
is a named storage location that contains data.

Example in real life:
  Box labeled "Age" contains the number 16
  Box labeled "Name" contains the text "Alex"

In JavaScript:
  const age = 16;
  const name = "Alex";


B. PRIMITIVE DATA TYPES (The Basic Building Blocks)
These are the simplest forms of data. Think of them as atoms - you can't break 
them down further.

  1. STRING - Text data (always in quotes)
     What it is: Words, sentences, characters, anything text-based
     Examples: "Hello", 'World', `Hello World`
     Why it matters: Storing names, messages, descriptions, etc.
     
  2. NUMBER - Any mathematical number
     What it is: Integers (whole numbers) or decimals
     Examples: 42, 3.14, -7, 0
     Why it matters: Storing ages, prices, quantities, measurements
     
  3. BOOLEAN - True or False (only 2 options)
     What it is: A yes/no or on/off switch
     Examples: true, false
     Why it matters: Making decisions in your code (if this is true, do that)
     
  4. UNDEFINED - Empty or no value
     What it is: A placeholder meaning "nothing is here yet"
     Example: let x; (nothing assigned, so it's undefined)
     Why it matters: Sometimes you need to mark a spot as "empty for now"


C. ARRAYS - Lists of Things
An array is like a shopping list - it holds multiple items in order.

What it looks like:
  const fruits = ["apple", "banana", "orange"];
  
Breaking it down:
  - [ ] = array container
  - "apple", "banana", "orange" = items in the list (each separated by comma)
  - Order matters! "apple" is at position 0, "banana" is at position 1, etc.

Why arrays are cool:
  - You can mix different types: [4, "two", true]
  - You can store numbers and text together
  - You can access items by their position (called an INDEX)


D. OBJECTS - Labeled Collections
An object is like a person's profile - it has different pieces of information 
with labels (called "properties" or "keys").

What it looks like:
  const student = {
    name: "Alex",
    age: 16,
    isStudent: true
  };

Breaking it down:
  - { } = object container
  - name, age, isStudent = labels (called "keys" or "properties")
  - "Alex", 16, true = the values stored under each label
  - Each pair is separated by a comma

Why objects are useful:
  - You can organize related information together
  - Each piece has a meaningful label
  - You can access info by name: student.name gives you "Alex"


E. NESTED STRUCTURES - Boxes Inside Boxes
Sometimes you need more complex structures. This is like having a box that 
contains smaller boxes!

Real-world example:
  A company is an object:
    - Name: "TechCorp"
    - Employees: [list of people]
    - Location: {city: "NYC", zip: "10001"}
                 ‚îî‚îÄ This is an object inside an object!

In code:
  const company = {
    name: "TechCorp",
    employees: ["Alice", "Bob", "Charlie"],  // Array inside object
    location: {                              // Object inside object
      city: "NYC",
      zip: "10001"
    }
  };


F. SPECIAL CHARACTERS & ESCAPE SEQUENCES
Some characters need special handling in strings because they have meaning 
in code.

Common special characters:
  ‚Ä¢ ` (backtick) - Used for special strings called template literals
  ‚Ä¢ \ (backslash) - Escapes special characters
  ‚Ä¢ / (forward slash) - Used in paths and patterns
  ‚Ä¢ " (double quote) - Marks string boundaries
  ‚Ä¢ ' (single quote) - Marks string boundaries

When you need these IN your string, you escape them with a backslash:
  ‚Ä¢ \" means a double quote character (not the end of the string)
  ‚Ä¢ \' means a single quote character
  ‚Ä¢ \\ means a backslash character itself
  ‚Ä¢ \` means a backtick character

Example:
  const message = "She said \"Hello!\"";
  // Result: She said "Hello!"


G. CONST - Constants (Unchangeable Variables)
"const" means "constant" - something that doesn't change.

Think of it like writing in permanent marker on a box:
  - You label it once
  - You can't change the label
  - Whatever you put in stays as is

In code:
  const age = 16;  // Set it once, can't change it
  age = 17;        // ERROR! Can't do this!

Why use const?
  - Prevents accidents (you can't accidentally change it)
  - Shows your intention: "This value is fixed"
  - Makes code safer and easier to understand


H. OBJECT.FREEZE - Locking Your Boxes
freeze() is like putting a box in a security case so nothing inside can be 
changed.

What it does:
  - Prevents adding new properties
  - Prevents removing properties
  - Prevents changing existing properties

Example:
  const student = {
    name: "Alex",
    age: 16
  };
  
  Object.freeze(student);  // Lock it down!
  
  student.name = "Jordan";  // ERROR! Can't change
  student.city = "NYC";     // ERROR! Can't add new property

Why freeze?
  - Protects important data from being modified
  - Shows: "This data should not change"
  - Makes code more predictable and secure

3. STEP-BY-STEP IMPLEMENTATION
================================================================================

STEP 1: SET UP YOUR FILE
Create a new file called "declarations.js" (or open an existing one)
This is where you'll write your code.


STEP 2: CREATE escapeStr (Text with Special Characters)
Goal: Store a string that contains: `, \, /, " and '

Why this matters: 
  You need to learn how to handle tricky characters that have special meaning 
  in code.

How to do it:
  const escapeStr = "`\\/\"'";
  
Breaking down the escaping:
  ‚Ä¢ ` = backtick (no escape needed if using single/double quotes)
  ‚Ä¢ \\ = this represents ONE backslash (you need two to show one)
  ‚Ä¢ \/ = this represents ONE forward slash (technically doesn't need escaping, 
        but shows best practice)
  ‚Ä¢ \" = this represents ONE double quote (escaped because the string is 
       wrapped in double quotes)
  ‚Ä¢ ' = single quote (no escape needed if string uses double quotes)

Result: When you look at this string, it contains 5 characters: ` \ / " '


STEP 3: CREATE arr (Simple Array)
Goal: Store an array with the number 4 and the string '2'

Code:
  const arr = [4, '2'];
  
Explanation:
  ‚Ä¢ First item: 4 (a number, no quotes)
  ‚Ä¢ Second item: '2' (text that LOOKS like 2 but is actually text)

Why the difference?
  ‚Ä¢ 4 is a NUMBER (can do math with it)
  ‚Ä¢ '2' is TEXT (just looks like a number, can't do math with it)

This teaches you: Arrays can hold DIFFERENT types of data!


STEP 4: CREATE obj (Object with Primitive Values)
Goal: Store an object with 4 different types of data

Code:
  const obj = {
    str: "I am a string",
    num: 42,
    bool: true,
    undef: undefined
  };

Explanation:
  ‚Ä¢ str: holds TEXT (a string)
  ‚Ä¢ num: holds a NUMBER
  ‚Ä¢ bool: holds TRUE or FALSE (a boolean)
  ‚Ä¢ undef: holds nothing/empty (undefined)

Why this matters: You're learning that objects can hold DIFFERENT types of 
information all organized together!


STEP 5: CREATE nested (Complex Structure with Boxes Inside Boxes)
Goal: Store an object that contains an array AND another object

Code:
  const nested = {
    arr: [4, undefined, '2'],
    obj: {
      str: "I am a string",
      num: 42,
      bool: true
    }
  };

Breaking it down:
  LEVEL 1 (outer object):
    ‚Ä¢ arr: an array with 3 items (4, undefined, and '2')
    ‚Ä¢ obj: another object (nested inside!)
    
  LEVEL 2 (inner object):
    ‚Ä¢ str: text
    ‚Ä¢ num: number  
    ‚Ä¢ bool: boolean

Visualization (like Russian nesting dolls):
  nested = {
    arr: [4, undefined, '2']
    obj: {
      str: ...
      num: ...
      bool: ...
    }
  }


STEP 6: FREEZE ALL THREE DATA STRUCTURES
Goal: Lock arr, obj, and nested so they can't be changed

Code:
  Object.freeze(arr);
  Object.freeze(obj);
  Object.freeze(nested);

Why each one?
  ‚Ä¢ freeze(arr): Prevents anyone from changing the items in the array
  ‚Ä¢ freeze(obj): Prevents anyone from changing properties in the object
  ‚Ä¢ freeze(nested): Prevents changes to the nested structure

After freezing:
  arr[0] = 99;           // ERROR! Can't change
  obj.str = "new text";  // ERROR! Can't change
  nested.arr[0] = 99;    // ERROR! Can't change

Note about freeze():
  freeze() only works on the TOP LEVEL!
  For nested, it freezes the outer object, but ideally you'd freeze the inner 
  object too:
  
  Object.freeze(nested.obj);
  
  This makes it completely safe!


STEP 7: VERIFY YOUR WORK
Quick test - your final code should look like this:

  const escapeStr = "`\\/\"'";
  const arr = [4, '2'];
  const obj = {
    str: "I am a string",
    num: 42,
    bool: true,
    undef: undefined
  };
  const nested = {
    arr: [4, undefined, '2'],
    obj: {
      str: "I am a string",
      num: 42,
      bool: true
    }
  };
  
  Object.freeze(arr);
  Object.freeze(obj);
  Object.freeze(nested);
  Object.freeze(nested.obj);  // Freeze inner object too!

Test it:
  ‚Ä¢ Try to change a value (it should fail or ignore the change)
  ‚Ä¢ Try to add a new property (it should fail or ignore it)
  ‚Ä¢ Log the values to console to see them

EXAMPLE 1: Working with Your Variables
// Access string
console.log(escapeStr);  // Output: `\/"'

// Access array items
console.log(arr[0]);     // Output: 4

// Access object properties
console.log(obj.str);    // Output: "I am a string"

// Access nested values

console.log(nested.obj.str);        // Output: "I am a string"


EXAMPLE 2: Trying to Modify (Will Fail)
// This will silently fail or throw an error (depending on strict mode)

obj.str = "Changed!";
nested.arr[1] = "modified";

// The values remain unchanged!
console.log(arr[0]);           // Still outputs: 4
console.log(obj.str);          // Still outputs: "I am a string"

6. COMMON MISTAKES & HOW TO AVOID THEM
================================================================================

MISTAKE 1: Forgetting to Escape Special Characters
‚ùå Wrong:
  const str = "She said "Hello"";  // ERROR! Quotes confuse the computer

‚úÖ Correct:
  const str = "She said \"Hello\"";  // Use backslash to escape


MISTAKE 2: Confusing String Numbers with Real Numbers
‚ùå Problem:
  const num1 = 5;    // This is a number - can do math
  const num2 = "5";  // This is text - looks like a number but it's not
  num1 + num2;       // Gives "55" (combines text!) not 10

‚úÖ Better:
  Use numbers for math, text only when needed


MISTAKE 3: Forgetting That freeze() Only Works One Level Deep
‚ùå Problem:
  const obj = { nested: { val: 10 } };
  Object.freeze(obj);
  obj.nested.val = 20;  // This WILL work! (not frozen)

‚úÖ Better:
  Object.freeze(obj);
  Object.freeze(obj.nested);  // Freeze the inside too!


MISTAKE 4: Trying to Modify a Const
‚ùå Wrong:
  const age = 16;
  age = 17;  // ERROR! Can't reassign a const

‚úÖ Correct:
  const age = 16;  // Keep it as is, or use let if it needs to change


MISTAKE 5: Forgetting Commas Between Object Properties
‚ùå Wrong:
  const person = {
    name: "Alex"
    age: 16        // Missing comma before age!
  };

‚úÖ Correct:
  const person = {
    name: "Alex",
    age: 16
  };
