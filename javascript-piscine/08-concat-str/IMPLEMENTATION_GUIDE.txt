PART 1: UNDERSTANDING THE ASSIGNMENT
================================================================================

What does "concat" mean?
------------------------
The word "concat" is short for "concatenate," which means to join or link things
together. Think of it like connecting train cars - you have two separate cars 
(strings in our case), and you link them together to make one long train.

What is your task?
------------------
You need to create a function named "concatStr" that:
  1. Takes 2 arguments (inputs) as parameters
  2. Joins them together into one result
  3. Returns the combined result

Simple Example:
  If you give the function:
    - First input: "Hello"
    - Second input: "World"
  
  The function should return: "HelloWorld"

================================================================================
PART 2: KEY CONCEPTS YOU NEED TO KNOW
================================================================================

A) WHAT IS A FUNCTION?
----------------------
A function is like a recipe or instruction manual. It takes inputs (ingredients),
performs actions (cooking steps), and produces an output (the final dish).

Think of it like a pizza-making machine:
  - INPUT: dough and toppings
  - PROCESS: mix, bake
  - OUTPUT: finished pizza

In JavaScript, we create functions so we can reuse the same code over and over
without writing it multiple times.


B) WHAT IS A PARAMETER/ARGUMENT?
---------------------------------
A parameter is a variable (placeholder) that receives input when you call
a function. An argument is the actual value you pass to the function.

Example analogy:
  - PARAMETER: Like a label on a box that says "put your name here"
  - ARGUMENT: Like actually writing your name in that box


C) WHAT IS STRING CONCATENATION?
---------------------------------
A string is text - like "Hello" or "JavaScript". Concatenation is joining
multiple strings together.

Methods to concatenate strings in JavaScript:

  Method 1: Using the + operator (simplest)
  ---------
  result = "Hello" + " " + "World";  // Result: "Hello World"

  Method 2: Using template literals (modern and clean)
  ---------
  result = `Hello World`;  // Using backticks and ${variable}
  OR
  result = `${str1}${str2}`;

  Method 3: Using the .concat() method (object method)
  ---------
  result = "Hello".concat(" ", "World");  // Result: "Hello World"


D) RETURN STATEMENT
-------------------
The "return" keyword tells the function what value to send back to whoever
called the function. Without it, the function won't give you any result!

Example:
  function add(a, b) {
    return a + b;  // This sends the sum back
  }
  
  let result = add(5, 3);  // result will be 8
  console.log(result);     // prints: 8

PART 3: STEP-BY-STEP BREAKDOWN
================================================================================

Step 1: Write the Function Declaration
---------------------------------------
We start by telling JavaScript we're creating a function:

  function concatStr(str1, str2) {
    // code goes here
  }

Explanation:
  - "function" = keyword that says "I'm making a function"
  - "concatStr" = the name of our function
  - (str1, str2) = parameters - two placeholders for our inputs


Step 2: Add the Logic (The Action)
----------------------------------
Inside the function, we need to join the two strings together:

  function concatStr(str1, str2) {
    return str1 + str2;
  }

Explanation:
  - "str1 + str2" = adds (concatenates) the two strings together
  - "return" = sends the combined result back out


Step 3: How to Use It
---------------------
Once the function is defined, you can call it like this:

  concatStr("Hello", "World");     // Returns: "HelloWorld"
  concatStr("JavaScript", " is awesome");  // Returns: "JavaScript is awesome"
  
You can also store the result in a variable:

  let result = concatStr("Code", "Master");
  console.log(result);  // Prints: "CodeMaster"

PART 4: IMPORTANT CONCEPTS TO REMEMBER
================================================================================

1. NAMING CONVENTIONS
   - Function names should be meaningful (concatStr clearly tells us what it does)
   - Use camelCase: first word lowercase, next words capitalized
   - Examples: concatStr, addNumbers, isUserValid

2. PARAMETERS VS ARGUMENTS
   - Parameters are defined when you create the function: function concatStr(str1, str2)
   - Arguments are the values you pass when you call it: concatStr("Hello", "World")

3. THE + OPERATOR WITH STRINGS
   - When you use + between strings, it concatenates them
   - "Hello" + " " + "World" = "Hello World"
   - Order matters! "Hello" + "World" ‚â† "World" + "Hello"

4. RETURN IS CRUCIAL
   - Without return, your function won't give you any result
   - The function stops executing once it hits return
   - You can only return ONE value (but it can be complex)

Below is the complete, ready-to-use code for this assignment:

// Function Declaration: This is the main solution
function concatStr(str1, str2) {

================================================================================
ALTERNATIVE SOLUTIONS (For Learning)
================================================================================

Here are other ways to solve this same problem. They all work!

ALTERNATIVE 1: Using Template Literals (Modern JavaScript)
-----------------------------------------------------------
function concatStr(str1, str2) {
  return `${str1}${str2}`;
}

Advantage: More modern syntax, easier to read with variables


ALTERNATIVE 2: Using the .concat() Method
-------------------------------------------
function concatStr(str1, str2) {
  return str1.concat(str2);
}

Advantage: Explicitly uses the concat method (matches the function name!)


ALTERNATIVE 3: Using .join()
-----------------------------
function concatStr(str1, str2) {
  return [str1, str2].join('');
}

Advantage: Good for combining many strings (beyond just 2)


ALTERNATIVE 4: Arrow Function (Shorter Syntax)
-----------------------------------------------
const concatStr = (str1, str2) => str1 + str2;

Advantage: More concise, modern JavaScript style


All four solutions produce the same result. As you learn more, you'll discover
that there are often multiple ways to solve the same problem in programming!
