\section{Conclusion}

\par Amazon S3 (Simple Storage Service) and the AWS SDK for Java 2.x were successfully 
utilized to develop and deploy cloud-based storage solutions across multiple geographic 
regions. This lab demonstrated the fundamental concepts of cloud storage architecture, 
multi-region deployment strategies, and performance optimization through hands-on 
implementation. By developing six comprehensive Java applications covering bucket creation, 
object management, and latency measurement, the group gained practical experience with 
the builder pattern, regional configuration, credential management, and REST API 
interaction with cloud services. The implementation and testing of these applications 
confirmed that AWS S3 provides reliable, scalable, and geographically distributed storage 
capabilities suitable for enterprise applications. Through this exercise, the team 
strengthened their understanding of cloud infrastructure design patterns, the importance 
of region-specific deployment considerations, and how network latency directly impacts 
user experience in distributed systems.

\par The latency measurements across three European regions (Stockholm, Ireland, and 
Frankfurt) provided quantitative evidence of real-world cloud performance characteristics. 
The results demonstrated that regional proximity significantly influences application 
performance, with upload/download operations showing 7-8\% variation between the fastest 
and slowest regions. This data-driven analysis reinforces the architectural principle 
that cloud applications must carefully consider geographic region selection to optimize 
for their user base and compliance requirements. The successful implementation of 
idempotent operations (such as bucket deletion) and robust error handling demonstrates 
the resilience principles essential for production cloud applications. Additionally, the 
integration of the AWS SDK, Maven dependency management, and proper resource lifecycle 
management (using try-with-resources patterns) illustrates industry best practices for 
cloud development. This lab provided a foundation for understanding how modern 
applications leverage cloud storage services to achieve scalability, reliability, and 
geographic distribution at scale.
