PART 3: HOW TO TAKE SCREENSHOTS
===========================================================================

METHOD 1: Linux (Gnome/KDE)

Using Print Screen:
```bash
# Take full screen:
Print key

# Take area:
Shift + Print

# Take window:
Alt + Print

# Delay screenshot:
print-screen-app --delay=5
```

Using gnome-screenshot:
```bash
# Full screen:
gnome-screenshot

# Specific window:
gnome-screenshot -w

# Delay:
gnome-screenshot --delay=5
```

Using screengrab:
```bash
# Install:
sudo apt-get install screengrab

# Run:
screengrab
```

---

METHOD 2: Windows/Mac

Windows (Build-in):
- Print Screen key (full screen)
- Alt + Print Screen (active window)
- Shift + Windows Key + S (region selection)

Mac (Build-in):
- Cmd + Shift + 3 (full screen)
- Cmd + Shift + 4 (region selection)
- Cmd + Shift + 5 (recording)

---

METHOD 3: Terminal Capture

Using `script` command:
```bash
# Start recording terminal:
script output.txt

# Run your commands:
mvn exec:java -Dexec.mainClass="..."

# End recording:
exit

# View output:
cat output.txt
```

Using `asciinema`:
```bash
# Install:
pip install asciinema

# Record:
asciinema rec demo.cast

# Run commands...

# Stop with Ctrl+D

# Convert to screenshot (with tools like ffmpeg)
```

---

METHOD 4: IDE Screenshots

If using VS Code:
1. Open terminal within VS Code
2. Run Maven command
3. Take screenshot of entire VS Code window
4. Should show: editor on left, terminal output on right

PART 7: WHAT NOT TO INCLUDE
===========================================================================

‚ùå NEVER INCLUDE:
- AWS secret keys or access keys
- API keys or tokens
- Passwords or credentials
- Personal information (phone, email, address)
- System paths that reveal personal info
- IP addresses (if static/identifying)
- AWS account numbers (if you want to hide identity)

IF ACCIDENTAL INCLUSION:
1. Redact using image editor (black rectangle over sensitive data)
2. Take screenshot again
3. Make sure no sensitive data visible in final version

BEFORE SUBMITTING:

```bash
# Check for sensitive data:
grep -r "AKIA" Screenshots/
# Should return nothing (no AWS keys)

grep -r "secret" Screenshots/
# Should return nothing

grep -r "password" Screenshots/
# Should return nothing
```

PART 8: IMAGE OPTIMIZATION
===========================================================================

FILE SIZE CONSIDERATION:

PNG files can be large. To optimize:

Using ImageMagick:
```bash
# Convert and compress:
convert input.png -resize 1920x1080 -quality 85 output.png

# Or:
mogrify -quality 85 *.png
```

Using pngquant:
```bash
# Install:
sudo apt-get install pngquant

# Compress:
pngquant 256 input.png --output output.png
```

Using optipng:
```bash
# Install:
sudo apt-get install optipng

# Optimize:
optipng -o7 input.png
```

QUALITY BALANCE:
- For terminal output: PNG at quality 85-90 is fine
- Readable text maintained: minimum font size 10pt
- File size: typically 100-500 KB per screenshot
- Total for 12 screenshots: 1-5 MB
