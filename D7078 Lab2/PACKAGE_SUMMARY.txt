WHAT EACH SCRIPT/PROGRAM DOES
================================================================================

task-2-1-setup.sh - Creates AWS Infrastructure
   Steps:
   1. Checks AWS CLI installation
   2. Creates Security Group with rules
      - SSH only from your IP
      - HTTP from anywhere (0.0.0.0/0)
   3. Creates S3 bucket
   4. Creates IAM policy (Lab2S3Policy.json)
   5. Creates IAM role (Lab2S3AccessRole)
   6. Attaches policy to role
   7. Creates instance profile
   8. Creates EC2 key pair
   9. Launches EC2 instance
   10. Saves configuration to lab2-config.txt
   
   Output:
   - lab2-keypair.pem (EC2 key)
   - Lab2S3Policy.json (policy definition)
   - lab2-config.txt (configuration for cleanup)

task-2-1-cleanup.sh - Removes All Resources
   Steps:
   1. Terminates EC2 instance
   2. Deletes security group
   3. Detaches and deletes IAM policy
   4. Deletes IAM role and instance profile
   5. Deletes S3 bucket
   6. Removes local configuration files
   
   Safety: Asks for confirmation before proceeding

S3BucketOperations.java - S3 CRUD Operations Demo
   Tasks:
   1. Creates buckets in 3 regions (us-east-1, eu-west-1, ap-southeast-1)
   2. Lists all buckets
   3. Uploads objects to buckets
   4. Downloads objects from buckets
   5. Lists objects within bucket
   6. Deletes objects
   
   Output: Console messages showing each operation

S3LatencyMeasurement.java - Performance Testing
   Tasks:
   1. Tests latency in each of 3 regions
   2. Creates 1 MB test files
   3. Uploads and measures latency (3 iterations)
   4. Downloads and measures latency (3 iterations)
   5. Calculates averages
   6. Displays results table
   7. Cleans up test buckets
   
   Output: Formatted latency results table
