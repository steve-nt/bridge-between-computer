================================================================================
HOW TO DECREASE INSTANCES FASTER
================================================================================

Default scale-in: 10-15 minutes (requires 5 minutes below 30%)

You can reduce this to 2-3 minutes with these changes:

================================================================================
METHOD 1: LOWER THE SCALE-IN THRESHOLD (EASIEST)
================================================================================

Current: Scales in when CPU < 30%
Faster: Scales in when CPU < 50%

Effect: Will remove instances much faster after load stops

Steps:

1. Go to: CloudWatch > Alarms > All Alarms

2. Find: "cpu-low-alarm-30percent"
   (or similar - look for scale-IN alarm, not scale-OUT)

3. Click on it

4. Click: "Edit"

5. Change:
   FROM: Threshold = 30%
   TO:   Threshold = 50%

6. Save changes

7. Now instances will scale down when CPU < 50%
   (Instead of waiting for < 30%)

Result: Much faster scale-in! âœ“

Caution: May be too aggressive for production
         During normal work, CPU might drop to 45% temporarily
         Could cause unwanted scale-in

RECOMMENDED FOR LAB: Yes (want to see it work quickly)

Faster: Needs only 2 minutes below threshold

Steps:

   TO:   60 seconds
